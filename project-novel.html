01 · Definition
需求定义与目标分析
项目定义

针对新用户 / 低活跃的激励敏感小说用户，通过首次进入时的新手引导流程，解决用户不知道「看小说可以领权益」的认知断层问题，达成阅读时长提升和留存改善的目标。

核心目标

通过新手引导打通用户对「看小说 = 赚钱」的认知链路，驱动阅读时长、用户活跃度和留存提升。

⚠ 核心设计矛盾

阅读沉浸感 vs 激励引导的力度是本需求核心的设计矛盾。小说阅读是高专注度的内容消费行为，过度的弹窗 / 动效干扰会直接损害内容体验，导致用户反感。这与短视频场景（低专注度、碎片化）有本质差异，不能完全照搬短视频的引导强度。设计策略需要在「让用户感知到激励」和「不破坏阅读心流」之间找到平衡点。

目标维度 当前状态 设计可影响的路径 风险
阅读时长提升 无引导，激励感知为零 即时奖励反馈 → 驱动翻页行为 引导过重会打断沉浸阅读体验
激励兑换转化 用户不知道任务页入口 引导路径串联三个页面 路径过长导致中途流失
7日留存提升 无激励钩子 阶段性任务 → 建立回访动机 激励预算不足时用户期望落差大
02 · Problem Analysis
现状问题分析
核心结论

设计机会点：激励敏感用户的核心诉求是「确认赚钱是真的、知道怎么赚、感受到在赚」，引导设计需要在这三个认知节点上分别给出明确信号。

问题 1

激励机制无预期

  • 用户进入读书页，看不到任何「看小说可以赚钱」的信号
  • 右上角金豆图标表意模糊，动效反馈不及时，激励感知弱
问题 2

翻页反馈表意模糊

  • 右上角圆圈进度图标反馈机制模糊，难理解
问题 3

提现门槛高且复杂

  • 首次提现门槛：翻 n 页 + 每页停留 8 秒 + 连续 2 天 + 翻页积攒到 0.3 元
  • 前 2 个任务直接提现，后面的任务是攒金豆 → 换算现金
  • 任务页信息密度高，新用户进来容易迷失，不知道从哪里开始
  • 任务页阅读提现任务副标题文案与实际不符，用户易被迷惑
问题 4

路径断点多，流程不清晰

  • 读书页 → 任务页 → 提现页三个页面之间没有主动串联
  • 用户不知道要点哪里、做什么、做完能得到什么
  • 任务页提现入口「可兑 0.2 元」，进入提现又写 0.3 元，信息矛盾
用户场景与体验链路问题排查
场景 1 首次进入

首次进入读书页，完全不知道有激励

新用户 / 低活跃用户首次打开一本小说进入读书页，动机是纯内容消费，对激励体系无感知。行为路径：进入读书页 → 开始翻页阅读 → 不知道右上角金豆入口是什么 → 正常阅读或直接退出。

首次进入的第一个翻页动作是最佳的激励感知植入时机，此时用户注意力集中、行为意图明确。
场景 2 知道激励

知道有激励，但不知道怎么兑换

用户看到了金豆数字，但不理解金豆的价值。行为路径：点击右上角金豆图标 → 进入任务页（弹窗形式）→ 看到「提现 / 书券 / 美团币」三个选项 → 发现金豆不足，无法提现 → 看到「阅读提现任务」需要翻页才能解锁 → 理解了规则，但路径较长。

任务页的信息层级需要让用户一眼看懂「我现在能做什么」，当前页面信息密度较高，新用户容易迷失。
场景 3 有阅读习惯

有阅读习惯，但不知道有任务体系

低活跃老用户，偶尔看小说但从未做过任务。被激励引导触达后，想了解能赚多少。行为路径：被引导弹窗吸引 → 进入任务页 → 看到「连看 2 天可提现 0.3 元」→ 产生「明天还要来」的回访动机。

阶段性提现门槛的可视化进度是建立回访习惯的关键，需要让用户清晰感知「我离下一个奖励还差多少」。
场景 4 主动探索

激励敏感用户主动探索提现

用户已积累一定金豆，想提现，确认钱是真的能到账的。行为路径：进入提现页 → 看到「连看 2 天可提现 0.3 元」（当前已选中）→ 发现「金豆不足」→ 返回读书页继续阅读。

提现页的「金豆不足」状态需要给出明确的行动指引，而不是让用户自己想办法。
理想体验链路
首次进入读书页
触发点①翻第 1 页后 → 浮现轻量引导提示：「翻页可赚金豆，看越多赚越多」
用户继续翻页（被激励驱动）
触发点②翻第 5 页 → 金豆到账动效 + 「已赚 X 金豆」即时反馈
触发点③引导点击金豆入口 → 进入任务页
任务页：清晰展示「我现在能做什么」+ 最近可达成的提现目标
触发点④引导「去完成任务」→ 返回读书页继续阅读
复访钩子次日推送：「你的阅读任务还差 X 页，今天完成可提现」
关键断点识别

当前链路中，触发点①②③完全缺失，用户从进入读书页到理解激励体系需要完全依靠自主探索，这对激励敏感用户来说是不可接受的认知成本。

03 · Competitive Research
竞品调研
核心结论

美团短视频的激励引导模式高度可参考，核心是「即时反馈 + 阶段进度 + 低打扰」三层设计，但小说场景需要降低引导强度以保护阅读心流。

短视频的「进入引导」提供了前置预期，让用户建立了看视频能赚钱的认知；短视频的「悬浮挂件」持续给用户一个「我在赚钱」的心理锚点。这个设计逻辑适用于小说场景——读书页右上角的金豆图标已经存在，但它是静态的、无反馈的，缺少「正在赚钱」的动态感知。
维度 美团短视频 番茄小说（间接竞品） 美团小说（现状）
首次进入引导 进入后自动弹出「看视频赚金币」引导弹窗，明确告知规则 无引导 无任何引导
即时反馈机制 刷视频过程中有「小财神」挂件悬浮，实时显示金币累积 无此类机制 无即时反馈
进度可视化 任务面板有进度条，清晰显示「还差 X 个视频完成任务」 无此类机制 任务页有进度但入口隐蔽
提现路径 金币 → 提现路径清晰，首页有显眼入口 无此类机制 提现页完整但入口需主动发现
引导强度 较强（弹窗 + 悬浮挂件 + 任务面板） 无 无
04 · Design Strategy
设计策略
核心结论

采用「轻引导 + 即时反馈 + 阶段钩子」三层策略，在不破坏阅读心流的前提下，建立用户对激励体系的认知和使用习惯。

5 个策略的关系
进入读书页
策略 1给预期 → 知道「这里能赚钱」
策略 2建信任 → 快速拿到第一个奖励，相信「是真的」
策略 3打通路径 → 知道「怎么赚、去哪赚、赚完去哪」，不迷路
策略 4赚钱心智 → 阅读中持续感知「我在赚」，愿意继续翻页
策略 5留存心智 → 「明天来能解锁更多」，主动回来
策略 3 是结构性的基础，解决的是流程能不能走通的问题；策略 1、2、4、5 是在全流程上，分别在不同节点强化用户的动机和信心。
Strategy 01

给激励预期

用户首次进入读书页时，主动用一句话说清楚利益点：「翻页就能赚金豆，金豆可以提现」。让用户在开始阅读的第一刻就建立「看小说 = 有收益」的预期。

Strategy 02

建立信任

新手引导阶段设置一个首次奖励（如翻 5 页即可领取），让用户快速完成「赚到」的体验，建立「这是真实可得的」信任感。

Strategy 03

建立赚钱心智

翻页时有轻量金豆到账动效，金豆数字实时更新，任务进度条清晰可见。让「赚钱感」贯穿整个阅读过程，而不是只在进入和退出时出现。

Strategy 04

打通路径，消除断点

读书页翻页后金豆到账，主动提示「去任务页看进度」；任务页完成任务后，主动引导「去提现页兑换」；提现页金豆不足时，告诉用户「还差 X 页，回去读就能解锁」。

Strategy 05

强化留存心智

将「连续阅读」机制前置到读书页：用户完成当天阅读后，明确告知「已完成第 1 天，明天继续可解锁 0.3 元提现」，让用户带着明确期待离开。

路径优化

简化核心流程

同时任务页的信息层级需要为新用户简化，突出「现在能做什么」，收起对新用户无用的复杂信息，每个页面都知道用户从哪来、该去哪，形成闭环。

改前路径

翻 n 页 → 任务页 → 点「领取」→ 提现页（不能提）→ 回到读书页 → 继续翻页 → 第 2 天继续... → 任务页 → 提现

改后路径

翻 n 页 → 任务页 → 点「领取」→ 提现页「提现」
